Java Tags / Keywords Meaning Back

  • abstract — Declares an abstract class or method.
  • assert — Used for debugging purposes to test assumptions.
  • boolean — Data type for true/false values.
  • break — Exits a loop or switch statement.
  • byte — 8-bit integer data type.
  • case — Defines a branch in a switch statement.
  • catch — Handles exceptions in try/catch blocks.
  • char — Single 16-bit Unicode character type.
  • class — Declares a class.
  • continue — Skips current iteration of a loop.
  • default — Default branch in a switch statement.
  • do — Executes a loop at least once.
  • double — 64-bit floating-point number.
  • else — Conditional branch if `if` condition is false.
  • enum — Defines a set of named constants.
  • extends — Indicates inheritance.
  • final — Declares constants or prevents inheritance/overriding.
  • finally — Executes a block after try/catch, always.
  • float — 32-bit floating-point number.
  • for — Loop with initialization, condition, and increment.
  • if — Conditional statement.
  • implements — Implements an interface.
  • import — Includes classes or packages.
  • instanceof — Tests object type.
  • int — 32-bit integer type.
  • interface — Declares an interface.
  • long — 64-bit integer type.
  • native — Declares native method implemented in other languages.
  • new — Creates new objects.
  • null — Represents null reference.
  • package — Declares package for a class.
  • private — Access modifier (class-only visibility).
  • protected — Access modifier (package + subclass visibility).
  • public — Access modifier (anywhere visibility).
  • return — Returns value from a method.
  • short — 16-bit integer.
  • static — Member belongs to class, not instance.
  • strictfp — Restricts floating-point calculations for portability.
  • super — Refers to parent class or constructor.
  • switch — Multi-way branch.
  • synchronized — Ensures thread-safe access.
  • this — Refers to current object.
  • throw — Throws exception.
  • throws — Declares exceptions a method can throw.
  • transient — Prevents serialization of a variable.
  • try — Starts exception handling block.
  • void — Method returns nothing.
  • volatile — Variable can be modified asynchronously.
  • while — Loop that continues while condition is true.
