STL Vector

vector is a resizable array in C++.
- push_back() adds elements.
- Works like an array but size can grow dynamically.

#include <iostream> #include <list> using namespace std; int main() { // List is a doubly linked list list<string> fruits = {"Apple", "Banana"}; // Adding elements at both ends fruits.push_front("Mango"); fruits.push_back("Orange"); // Iterating for (const auto &f : fruits) { cout << f << " "; } return 0; }

STL List

list is a doubly linked list.
- push_front() inserts at the beginning.
- push_back() inserts at the end.
- Efficient for insertion/removal anywhere.

Mango Apple Banana Orange

STL Set

set stores only unique values in sorted order.
- Duplicates are automatically removed.
- Useful for fast searching and uniqueness.

1 2 3 4
#include <iostream> #include <map> using namespace std; int main() { // Map stores key-value pairs map<string, int> age; age["Alice"] = 20; age["Bob"] = 25; // Access by key cout << "Alice is " << age["Alice"] << " years old." << endl; // Iterating for (auto &p : age) { cout << p.first << " : " << p.second << endl; } return 0; }

STL Map

map stores data in key-value pairs.
- Keys are unique.
- Values are accessed using the key.
- Internally uses a balanced binary tree.

Alice is 20 years old.

#include <iostream> #include <vector> using namespace std; int main() { vector<int> nums = {10, 20, 30}; // Using iterator vector<int>::iterator it; for (it = nums.begin(); it != nums.end(); ++it) { cout << *it << " "; } return 0; }

STL Iterators

iterator acts like a pointer for containers.
- begin() points to the first element.
- end() points after the last element.
- *it gives the value the iterator points to.

10 20 30
#include <iostream> #include <vector> #include <algorithm> using namespace std; int main() { vector<int> nums = {5, 2, 8, 1}; // Sort the vector sort(nums.begin(), nums.end()); // Find element if (binary_search(nums.begin(), nums.end(), 5)) { cout << "5 found!" << endl; } for (int n : nums) { cout << n << " "; } return 0; }

STL Algorithms

algorithm library provides ready-to-use functions.
- sort() sorts the elements.
- binary_search() searches efficiently in sorted containers.

#include <iostream> #include <memory> using namespace std; int main() { // unique_ptr: owns an object exclusively unique_ptr<int> ptr = make_unique<int>(10); cout << "Value: " << *ptr << endl; // Can't copy, only move ownership unique_ptr<int> ptr2 = move(ptr); if (!ptr) cout << "ptr is now empty" << endl; return 0; }

unique_ptr

unique_ptr manages a single object.
- Only one owner at a time.
- Automatically deletes memory when owner goes out of scope.

Value: 10
ptr is now empty
#include <iostream> #include <memory> using namespace std; int main() { // shared_ptr: multiple owners shared_ptr<int> p1 = make_shared<int>(42); shared_ptr<int> p2 = p1; // share ownership cout << "p1: " << *p1 << ", p2: " << *p2 << endl; cout << "Reference count: " << p1.use_count() << endl; return 0; }

shared_ptr

shared_ptr allows multiple smart pointers to share ownership.
- Keeps a reference counter.
- Deletes object when the last owner is gone.

p1: 42, p2: 42
Reference count: 2
#include <iostream> #include <memory> using namespace std; int main() { shared_ptr<int> sp = make_shared<int>(100); weak_ptr<int> wp = sp; // weak reference cout << "Shared value: " << *sp << endl; cout << "Use count: " << sp.use_count() << endl; // weak_ptr needs lock() to access if (auto temp = wp.lock()) { cout << "Weak pointer locked: " << *temp << endl; } return 0; }

weak_ptr

weak_ptr is a non-owning reference to shared_ptr object.
- Prevents circular references.
- Use lock() to safely access the object.

Shared value: 100
Use count: 1
Weak pointer locked: 100
