Print & Println

// Print vs println example public class PrintExample { public static void main(String[] args) { // println prints text then moves to a new line System.out.println("Hello, World!"); // print prints text but stays on the same line System.out.print("This is "); System.out.print("on the same line."); } }
Explanation (how it runs)
  • JVM starts at main and executes statements top to bottom.
  • System.out.println() prints the argument then appends a newline character.
  • System.out.print() prints without adding a newline, so subsequent prints continue on the same line.
Compile: javac PrintExample.java → Run: java PrintExample
Hello, World! This is on the same line.

Variables & Data Types

public class VariablesExample { public static void main(String[] args) { int age = 20; // integer double gpa = 1.75; // floating point String name = "Alice"; // text boolean isStudent = true; // true/false System.out.println("Name: " + name); System.out.println("Age: " + age); System.out.println("GPA: " + gpa); System.out.println("Is a student? " + isStudent); } }
Explanation
  • Variables store values in memory; the type determines allowed values and operations.
  • String is an object for text; using + concatenates strings.
  • Java is statically typed: types are declared and checked at compile time.
Compile & run as usual. No input needed.
Name: Alice Age: 20 GPA: 1.75 Is a student? true

User Input (Scanner)

import java.util.Scanner; public class InputExample { public static void main(String[] args) { Scanner sc = new Scanner(System.in); // create Scanner System.out.print("Enter your name: "); String name = sc.nextLine(); // read full line System.out.print("Enter your age: "); int age = sc.nextInt(); // read integer System.out.println("Hello " + name + ", you are " + age + " years old."); sc.close(); // close scanner to free system resources } }
Explanation
  • Scanner reads from System.in (keyboard).
  • nextLine() reads an entire line; nextInt() reads an integer token.
  • Careful mixing nextInt() and nextLine() — you may need to consume the newline.
Example run: type inputs when prompted.
Enter your name: Alice Enter your age: 20 Hello Alice, you are 20 years old.

If - Else If - Else

import java.util.Scanner; public class IfElseExample { public static void main(String[] args) { Scanner sc = new Scanner(System.in); System.out.print("Enter grade (0-100): "); int grade = sc.nextInt(); if (grade >= 90) { System.out.println("Excellent"); } else if (grade >= 75) { System.out.println("Passed"); } else { System.out.println("Failed"); } sc.close(); } }
Explanation
  • Control flow chooses which block runs based on conditions.
  • Conditions are evaluated top to bottom; the first true branch runs.
  • Use else if for multiple disjoint ranges.
Enter grade (0-100): 82 Passed

Switch - Case

import java.util.Scanner; public class SwitchExample { public static void main(String[] args) { Scanner sc = new Scanner(System.in); System.out.print("Choose (1-3): "); int c = sc.nextInt(); switch (c) { case 1: System.out.println("One"); break; case 2: System.out.println("Two"); break; case 3: System.out.println("Three"); break; default: System.out.println("Out of range"); } sc.close(); } }
Explanation
  • switch matches a value to case labels.
  • Remember to use break to prevent fall-through.
  • Works with integer types, strings (since Java 7), enums, etc.
Choose (1-3): 2 Two

Loops (for, while, do-while)

public class LoopsExample { public static void main(String[] args) { // for loop for (int i = 1; i <= 3; i++) { System.out.println("For: " + i); } // while loop int j = 1; while (j <= 2) { System.out.println("While: " + j); j++; } // do-while int k = 1; do { System.out.println("DoWhile: " + k); k++; } while (k <= 2); } }
Explanation
  • for is preferred when you know iteration count.
  • while checks condition before the loop body.
  • do-while runs the body at least once then checks the condition.
For: 1 For: 2 For: 3 While: 1 While: 2 DoWhile: 1 DoWhile: 2

Arrays (1D, 2D & 3D, Dynamic)

// 1D Array String[] fruits = {"Apple", "Banana", "Mango"}; for (int i = 0; i < fruits.length; i++) { System.out.println("Fruit " + (i+1) + ": " + fruits[i]); } // 2D Array int[][] matrix = {{1,2,3}, {4,5,6}, {7,8,9}}; System.out.println("\nMatrix:"); for (int i = 0; i < matrix.length; i++) { for (int j = 0; j < matrix[i].length; j++) { System.out.print(matrix[i][j] + " "); } System.out.println(); } // 3D Array int[][][] cube = { {{1,2},{3,4}}, {{5,6},{7,8}} }; System.out.println("\n3D Cube:"); for (int i = 0; i < cube.length; i++) { for (int j = 0; j < cube[i].length; j++) { for (int k = 0; k < cube[i][j].length; k++) { System.out.print(cube[i][j][k] + " "); } System.out.print("| "); } System.out.println(); } // Dynamic Array using ArrayList import java.util.ArrayList; ArrayList list = new ArrayList<>(); list.add("Red"); list.add("Green"); list.add("Blue"); System.out.println("\nArrayList:"); for (String color : list) { System.out.println(color); }
Explanation
  • 1D array: simple list accessed by index.
  • 2D array: rows & columns; access with matrix[row][col].
  • 3D array: cube-like structure; access with cube[layer][row][col].
  • Dynamic arrays (ArrayList) can grow/shrink at runtime; use add() and loop via enhanced for.
  • length for arrays, size() for ArrayList.
Compile & run as usual.
Fruit 1: Apple Fruit 2: Banana Fruit 3: Mango Matrix: 1 2 3 4 5 6 7 8 9 3D Cube: 1 2 | 3 4 | 5 6 | 7 8 | ArrayList: Red Green Blue

Inheritance

// Parent class class Animal { void sound() { System.out.println("Animal makes a sound"); } } // Child class extends parent class Dog extends Animal { // override sound @Override void sound() { System.out.println("Dog barks"); } } public class OOPInheritance { public static void main(String[] args) { Animal a = new Animal(); Dog d = new Dog(); a.sound(); d.sound(); } }
Explanation
  • extends creates an "is-a" relationship; child inherits fields and methods.
  • Child can override behavior to provide a specific implementation.
Animal makes a sound Dog barks

Method Overriding

class Parent { void show() { System.out.println("Parent show"); } } class Child extends Parent { @Override void show() { System.out.println("Child show"); } } public class OOPOverride { public static void main(String[] args) { Parent p = new Child(); // reference type Parent, object type Child p.show(); // calls Child's overridden method } }
Explanation
  • When a subclass redefines a parent method with the same signature, that's overriding.
  • At runtime, the JVM calls the object's actual implementation (dynamic dispatch).
Child show

Polymorphism

// Polymorphism via inheritence and overridden methods class Shape { void draw() { System.out.println("Draw shape"); } } class Circle extends Shape { @Override void draw() { System.out.println("Draw circle"); } } public class OOPPolymorphism { public static void main(String[] args) { Shape s = new Circle(); s.draw(); // Circle's draw runs } }
Explanation
  • Polymorphism allows treating subclass objects as superclass references.
  • Useful for writing flexible, extensible code (e.g., collections of Shape objects).
Draw circle

Thread (extends Thread)

class MyThread extends Thread { private String name; MyThread(String name) { this.name = name; } @Override public void run() { for (int i = 1; i <= 3; i++) { System.out.println(name + " - count " + i); try { Thread.sleep(200); } catch (InterruptedException e) { /* ignore */ } } } } public class ThreadExample { public static void main(String[] args) { Thread t1 = new MyThread("T1"); Thread t2 = new MyThread("T2"); t1.start(); // starts new thread that runs run() t2.start(); } }
Explanation
  • Extending Thread lets you override run() to define thread work.
  • Call start() to create a new OS thread (or JVM-managed thread) — do not call run() directly.
  • Thread.sleep() pauses the current thread.
T1 - count 1 T2 - count 1 T1 - count 2 T2 - count 2 T1 - count 3 T2 - count 3 (Actual order may vary due to scheduling)

Runnable (implements Runnable)

class Worker implements Runnable { private String name; Worker(String name) { this.name = name; } @Override public void run() { for (int i = 1; i <= 3; i++) { System.out.println(name + " -> " + i); try { Thread.sleep(200); } catch (InterruptedException e) { } } } } public class RunnableExample { public static void main(String[] args) { Thread t1 = new Thread(new Worker("W1")); Thread t2 = new Thread(new Worker("W2")); t1.start(); t2.start(); } }
Explanation
  • Implementing Runnable is preferred when the class needs to extend another class.
  • Pass the Runnable to a Thread and call start().
W1 -> 1 W2 -> 1 W1 -> 2 W2 -> 2 W1 -> 3 W2 -> 3 (Execution order may differ)

Synchronization (synchronized)

class Counter { private int count = 0; public synchronized void increment() { count++; } public int get() { return count; } } public class SyncExample { public static void main(String[] args) throws InterruptedException { Counter c = new Counter(); Thread t1 = new Thread(() -> { for(int i=0;i<1000;i++) c.increment(); }); Thread t2 = new Thread(() -> { for(int i=0;i<1000;i++) c.increment(); }); t1.start(); t2.start(); t1.join(); t2.join(); System.out.println("Count = " + c.get()); } }
Explanation
  • synchronized ensures only one thread executes the method at a time for that object.
  • Without synchronization, concurrent increments may interleave and produce incorrect counts.
  • Use join() to wait for threads to finish before reading shared state.
Count = 2000 (If unsynchronized, value may be less due to race conditions)

Sleep & Join

public class SleepJoinExample { public static void main(String[] args) throws InterruptedException { Thread t = new Thread(() -> { try { System.out.println("Worker sleeping..."); Thread.sleep(500); // pause worker System.out.println("Worker done."); } catch (InterruptedException e) { } }); t.start(); System.out.println("Main waiting for worker to finish..."); t.join(); // wait until t finishes System.out.println("Main continues."); } }
Explanation
  • Thread.sleep(ms) pauses the current thread for a given time.
  • join() blocks the calling thread until the target thread finishes.
Worker sleeping... Main waiting for worker to finish... Worker done. Main continues.

Simple Performance Tip (StringBuilder vs String concatenation)

public class PerfExample { public static void main(String[] args) { // Bad: repeated String concatenation in loop (creates many objects) String s = ""; for (int i = 0; i < 1000; i++) { s += i; // creates new String each time } // Better: use StringBuilder StringBuilder sb = new StringBuilder(); for (int i = 0; i < 1000; i++) { sb.append(i); } String result = sb.toString(); System.out.println("Length: " + result.length()); } }
Explanation
  • String in Java is immutable — repeated concatenation creates many intermediate objects.
  • StringBuilder is mutable and more efficient for heavy string building.
Length: 2896 (Actual length depends on concatenated numbers; demonstrates result produced)
